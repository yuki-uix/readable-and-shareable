Select：让正确信息进来
— 三种召回方案的精度边界
核心框架
Select = 精度问题，不是索引覆盖问题
代码库 10 万行，Agent 只需要 500 行。三种方案 Embedding Index / AST 图谱 / Grep 各有精度边界，都有一个共同的缺陷：grep 返回 200 个匹配，只有 3 个是真正的函数调用点。LSP 用编译器逻辑直接回答「这个函数在哪里被调用」。

架构决策、业务术语、团队约定、已知的坑。
