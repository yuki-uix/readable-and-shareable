成本结构：钱花在哪里
— Prefill / Decode / Prompt Cache 原理
核心框架
Prefill 阶段 O(n²) 是成本核心，Prompt Cache 跳过它
每次 API 调用，所有 input token 先经过 Prefill（一次性并行处理），计算量 O(n²)。这是最贵的阶段。Prompt Cache 在前缀完全匹配时直接复用上次的 KV Cache，跳过整个 Prefill 重算。
两个阶段对比
Prefill 阶段
一次性处理全部 input
计算量O(n²)
费用（Opus 4.6）$5 / MTok
Cache 命中后$0.50 / MTok
副产品KV Cache
Decode 阶段
逐 token 生成 output
计算量O(n)× token 数
费用（Opus 4.6）$25 / MTok
GPU 利用率较低
复用 KV Cache只算新 token
LLM 语境下的三种 Cache
KV Cache
同一请求内部
缓存当前请求已算的 K/V
跨请求：不能
你控制：不能
账单影响：透明
Prompt Cache
跨请求复用 Prefill
前缀完全一致才命中
跨请求：能
你控制：部分
账单：直接降 input 费率

跳过整个 API 调用
