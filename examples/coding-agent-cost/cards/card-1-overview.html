怎么优化 Coding Agent 的成本
— Context Engineering 调研笔记
核心框架
Context Engineering：让 Agent 在正确时间获得正确上下文
Input token 占 API 成本的 93–99%。优化 Coding Agent 成本的核心不是减少使用，而是提升每个 token 的信息密度——让进来的信息对，让已有的信息精简，让独立的任务隔离。
四个操作方向
Select
让正确信息进来
精准召回相关代码，不把整个仓库塞进上下文
Compress
减少已有 token
预防膨胀或主动压缩，Cache 和压缩是对立的
Isolate
拆任务换模型
子任务独立上下文，便宜模型做简单事
Write
信息搬到文件
不需要时刻在场的信息存文件系统，按需读回
