# Skill: digest — 深度解读 × 可视化卡片

## 定位

**readable-and-shareable 个人工具包的核心 skill。**

输入：一篇值得分享的文章（URL / 文本 / 文件路径）
输出：一张基于你的深度解读的可视化分享卡片

不是提炼原文，而是**你对这篇文章的真实解读**——卡片内容来自解读，不是原文的压缩。

---

## 执行步骤

### Step 1 — 摄入

接受文章输入：
- URL → `WebFetch` 抓取正文
- 文件路径 → `Read` 读取
- 粘贴文本 → 直接使用

若文章超过 5000 字，默认全量分析，不跳过任何部分。

---

### Step 2 — 深度解读

**这是整个 skill 的核心。**

将解读结果写入 `examples/{slug}/digest.md`，同时在对话中展示完整内容。

#### `digest.md` 结构

```markdown
# Digest — {文章标题}

**来源**：{URL 或文件路径}  
**日期**：{YYYY-MM-DD}

---

## 核心主张

作者真正在说什么？用一句话表达——不是标题，不是话题，而是论点。
> 例："作者认为 Context Engineering 的本质不是省钱技巧，而是一种对 Agent 认知结构的重新设计。"

---

## 隐藏假设

这个论点成立，需要读者接受哪些前提？这些前提是否被明说？
（列出 2-3 条，每条一句话）

---

## So What

读完这篇文章，你的判断应该改变什么？
不是「你学到了什么」，而是「你之前的什么认知需要更新」。

---

## 盲点 / 反例

这个结论在什么情况下不成立？
作者有意或无意回避了什么问题？

---

## 更大的连接

这篇文章指向的是哪个更大的趋势或问题？
它和哪些已有的概念框架形成对话？
```

#### 写作标准

- **不是摘要**：不要复述原文说了什么，而是说出它意味着什么
- **有观点**：每个部分都应该有可被质疑的判断，而不是中性描述
- **够具体**：「隐藏假设」和「盲点」必须针对这篇文章，不能套用通用说法
- **篇幅**：整个 digest.md 约 300-500 字，像一篇有密度的读书笔记

---

### Step 3 — Review

展示 digest.md 内容后，询问：

```
这是我对这篇文章的解读。

有没有哪个判断你觉得不准，或者你想加入自己的角度？
告诉我，我调整后再生成卡片。

（觉得准确的话直接说「继续」）
```

**用户修正时**：更新 `digest.md`，重新展示被修改的部分，再次确认。  
**用户确认后**：进入 Step 4。

---

### Step 4 — 选择风格

基于 digest.md 的**解读性质**推荐信息组织方式，而不是文章结构。

| 解读性质 | 推荐风格 |
|---|---|
| 有强结论 + 行动建议 | 结论前置（ENTJ）|
| 概念之间有深层关联 / 矛盾 | 意义连接（INFJ）|
| 需要展示推导过程 | 推导草稿（INTP）|
| 多个闪光点 + 意义驱动 | 星座聚焦（ENFP）|
| 有争议 / 开放问题 | 辩证发散（ENTP）|
| 强规范 / 最佳实践 | 档案归纳（ISTJ）|
| 有成长弧线 / 给读者的信 | 成长路线图（ENFJ）|
| 感受驱动 / 叙事弧线 | 叙事弧线（INFP）|

推荐时用自然语言说明理由，以解读内容为依据：

```
根据解读，这篇文章的核心是「{核心主张的关键词}」，
建议用「{方法名}（{MBTI}）」——{一句话理由，指向 digest.md 中的具体判断}。

也可以选其他风格，告诉我。
```

用户确认风格后，询问配色：

```
配色：
1. warm  — 暖米色，微信 / 公众号首选
2. night — 深色高对比，科技内容
3. ink   — 纯白极简，印刷感

（不选则默认 warm）
```

---

### Step 5 — 生成卡片

基于 `digest.md` 生成图片卡片 HTML，保存至 `examples/{slug}/image-card-{style}-{color}.html`。

**内容来源规则：**
- 卡片内容来自 `digest.md`，不重读原文
- 标题 / 核心判断 = 你的解读语言，不是原文标题
- 描述 = 解读中的具体判断，不是原文摘要

读取参考文件：
1. `references/styles/{style}.md` — 签名视觉元素 + 设计 token
2. `references/design-system.md` — 全局视觉规范

---

### Step 6 — 自动截图

HTML 生成并保存后，立即执行：

```bash
node scripts/screenshot.mjs {html路径} --json
```

- 成功：用 `Read` 展示 PNG 预览
- 失败：展示 HTML 路径 + 手动截图说明

---

### Step 7 — 输出摘要

```
✅ 生成完成

  解读  → examples/{slug}/digest.md
  文件  → examples/{slug}/image-card-{style}-{color}.html
  截图  → examples/{slug}/image-card-{style}-{color}.png

  满意直接分享，不满意告诉我哪里要改。
```

---

## 边界说明

- 仅输出**图片卡片**，不生成思维导图 / Interactive HTML / 漫画
- 不支持多人使用场景，面向个人深度阅读工作流
- 截图工具：`scripts/screenshot.mjs`（本地 Chrome，不依赖 CDN）
