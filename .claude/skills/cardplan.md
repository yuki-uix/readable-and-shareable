# Skill: /cardplan — 卡片规划

## 定位

基于已有的 `digest.md`，决定出几张卡、每张卡讲什么、用什么视觉风格。

**不生成图片**，只输出一份可执行的卡片计划。

---

## 输入

需要先有 `examples/{slug}/digest.md`（由 `/digest` 生成）。

调用方式：
- `/cardplan` — 自动寻找当前 session 最近的 digest.md
- `/cardplan {slug}` — 指定 slug

---

## 执行

### Step 1 — 读取 digest.md

读取 `examples/{slug}/digest.md`，提取核心主张、So What、更大的连接。

---

### Step 2 — 复杂度评估

**判定为复杂文章（需要系列卡），若满足任意两条：**
- 原文字数 > 3000 字（中文）或 > 5000 words（英文）
- 包含 3 个以上独立成立的主题板块
- 覆盖多个工具对比 / 框架 / 决策维度

**若为普通文章**：单张卡，直接进入 Step 3 选风格。

---

### Step 3 — 卡片数推导（复杂文章）

**卡片数 = 独立成立的核心认知单元数量**

判断标准：每张卡必须满足「读了这一张，不需要其他张也能带走完整认知」。

推导步骤：
1. 列出 digest.md 中所有核心论点
2. 合并共享同一底层逻辑的论点（同根同源 → 一张卡）
3. 合并不足以独立成卡的碎片论点（太碎 → 归入最近的卡）
4. 剩余单元数 = 卡片数，**上限 6，下限 2**

提出拆分方案，每张卡必须说明独立成立的理由：

```
基于独立认知单元分析，建议拆成 {N} 张系列卡：

Card 1 · {主题名}
独立理由：{为什么读这一张不需要其他张}
核心问题：{这张卡回答什么}

Card 2 · {主题名}
独立理由：{...}
核心问题：{...}

确认这个拆分，或告诉我怎么调整？
```

用户确认后进入 Step 4。

---

### Step 4 — 视觉语言选择

**MBTI 决定整个系列的视觉语言（颜色 / 字体 / 质感），布局结构由每张卡的内容单独决定。**
系列卡只选一次 MBTI，全程统一。

基于 digest.md 的整体气质推荐：

| 文章气质 | 推荐 MBTI | 视觉语言关键词 |
|---|---|---|
| 定量分析 + 系统框架 + ROI 导向 | INTJ | 冷白、深海蓝、大数字锚点、权威感 |
| 有强结论 + 行动建议 | ENTJ | 深炭灰、金色、执行感 |
| 需要展示推导过程 | INTP | 米黄、草稿感、暖褐 |
| 多个闪光点 + 意义驱动 | ENFP | 星空黑、琥珀、星座散点 |
| 有争议 / 开放问题 | ENTP | 双色辩论、高对比 |
| 强规范 / 最佳实践 | ISTJ | 森林绿、米白、结构感 |
| 有成长弧线 / 旅程感 | ENFJ | 琥珀、路径节点 |
| 感受驱动 / 叙事弧线 | INFP | 赤陶、暖米、蜿蜒路径 |
| 实用决策工具 | ISTP | 靛蓝、决策树、工程感 |
| 温暖对比指南 | ISFJ | 鼠尾草绿、暖米、双栏 |

推荐后询问输出模式：
```
输出模式：
1. 手绘笔记 — AI 生成图片，sketch-note 美学
2. 编辑卡片 — HTML 截图，精准排版

（不选则默认 手绘笔记）
```

用户确认后进入 Step 5。

---

### Step 5 — 写 card-plan.md

为每张卡写 mini-brief（100-200 字），保存至 `examples/{slug}/card-plan.md`：

```markdown
# Card Plan — {文章标题}

**MBTI**：{类型}
**输出模式**：{手绘笔记 / 编辑卡片}
**卡片总数**：{N}

---

## Card 1 · {主题名}

**独立理由**：{...}
**核心问题**：{...}
**关键内容**：
- {要点 1}
- {要点 2}
- {关键数据 / 引用（如有）}

---

## Card 2 · {主题名}

...
```

---

## 输出

```
✅ 卡片计划完成

  card-plan.md → examples/{slug}/card-plan.md
  共 {N} 张卡 · {MBTI} 视觉语言 · {输出模式}

  下一步：运行 /cardgen {slug} 开始生成图片。
```

---

## 边界

- 只做规划，不生成图片
- 不修改 digest.md
- 依赖 digest.md 存在，否则提示先运行 /digest
