Token 一致性的系统模型
三轴约束框架 → 摩擦来源的精确定位
核心框架
Token 一致性 = 三轴同向
品牌意图 · 认知任务 · Token 系统 — 三个约束来源的投票方向必须一致。
任意一轴偏移 → 交集消失 → 用户感知摩擦。
轴 A
品牌意图
传达什么性格 · 视觉语言的情绪方向
轴 B — 关键约束
认知任务
扫描 / 确认 / 阅读 / 感受品牌 · 由 IA 结构决定
轴 C
Token 系统
颜色 · 圆角 · 间距 · 字重 · 动效的完整集合
场景IA 约束认知任务自由度合法 Token 组合
Landing page 约束 → 0 认知任务 = Token 表达 9 种均合法
功能性 IA 有功能约束 认知任务独立于 Token 三轴交集非空时合法
三轴偏移 — 任意轴偏移 交集消失 → 摩擦
逻辑关系
认知任务 → 约束 Token 表达方向
IA 结构 → 决定 认知任务类型
三轴偏移 → ≠ 交集消失 → 「感觉不对」
∴ 演绎结论
Token 一致性不是审美判断，是三轴约束的结构结果
「感觉对了」= 品牌意图 · 认知任务 · Token 系统三轴同向投票，交集非空
