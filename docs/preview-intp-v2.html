Token 一致性：命题的第一原理拆解
// 初始命题（待检验）
Token 要跟品牌保持一致
「一致」是未定义项，命题无法验证
①
拆解「一致」：一致 ≠ 相似，而是三轴同向投票
品牌意图 · 认知任务 · Token 系统 — 三个独立变量
引入关键变量
★
认知任务 — 关键约束变量
由 IA 结构决定，独立于 Token 系统之外。
扫描 / 确认 / 阅读 / 感受品牌 → 不同任务需要不同投票方向
代入特殊情形
②
反例检验：Landing page — IA 约束 → 0
∴ 认知任务 = Token 表达本身，9 种组合均合法
∴ 「一致性」在此处约束消失 — 命题范围缩小
成立条件： 认知任务已知 IA 有功能约束 三轴同向 Landing page
∴ 条件结论
「感觉对了」= 三轴交集非空的结构结果
当 IA 存在功能约束 ∧ 三轴投票方向已知 时成立
