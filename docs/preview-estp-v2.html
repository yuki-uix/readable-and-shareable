TOKEN BRIEF
局面
Token 选错了
不是审美问题——
是约束没对齐
关键变量
认知任务 — 由 IA 结构决定，不由设计师决定
用户在这个界面要做什么 → 这个答案约束了所有 Token 的合法范围
NOW
01
问：用户的认知任务是什么？
扫描 / 确认 / 阅读 / 感受品牌
02
对照三轴找偏移点
品牌意图 · 认知任务 · Token
NEXT
03
三轴同向 → 执行 Token 决策
交集非空 = 合法选择
04
有摩擦 → 回到 01 重查
不要靠直觉调整，找约束源
典型摩擦场景
数据看板
用了「温暖亲切」Token
认知任务 = 扫描效率，需冷色高对比
支付确认
用了「探索发现」Token
认知任务 = 快速确认，需强 CTA
技术文档
用了「品牌氛围」Token
认知任务 = 深度阅读，需高可读性
Landing page
强行对齐三轴
IA 约束 = 0，9 种均合法，放开
