Prompt Cache 的运作机制
API 请求含完整 input
每轮触发
Prefill读取所有 token
O(n²) 成本input 占 93–99%
↓ 如果前缀稳定
Cache Hit跳过 Prefill
价格 ÷10
实际写入只计算新增部分
输出 token成本相对小
↓ 缓存失效条件
压缩历史记录修改前缀 → 缓存失效
上下文超窗口截断 → 前缀变动
会话重启新会话无缓存
关键 Compress 和 Cache 互相矛盾——压缩省 token 但破坏前缀稳定，导致缓存失效反而更贵。两者不能同时最大化。
