Token 一致性的操作流程
按步骤检查三轴是否同向，确认后再执行 Token 决策
共 4 步 · 建议在设计评审前完成
1
明确认知任务
已完成
确认用户在这个页面的主要任务是什么：扫描信息、完成操作、感受品牌，还是阅读详细内容？认知任务不同，Token 的合法范围完全不同。
认知任务已写入设计说明
2
确认品牌意图
已完成
这个界面要传达什么性格？专业 / 亲切 / 高效 / 创意——品牌意图决定了 Token 的情绪方向，需要在选 Token 之前就对齐。
品牌意图与设计团队对齐
3
检查三轴是否同向
进行中
品牌意图 · 认知任务 · Token 系统 — 三个轴的投票方向是否一致？将每个 Token 决策对照三轴逐项核对，找出偏移点。
注意：Landing page 是例外
Landing page 没有独立的 IA 功能约束，认知任务 = Token 表达本身。此时 9 种 Token 组合均合法，无需强行对齐三轴。
4
执行 Token 决策
确认三轴同向后，从 Token 系统中选取满足条件的集合。交集非空 = 合法选择。有摩擦感 = 回到步骤 3 重新检查。
✅
三轴同向 → Token 决策可执行
完成以上 4 步后，「感觉对了」是可预期的结构结果
