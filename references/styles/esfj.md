# ESFJ Style — 关系关怀图

**认知偏好**：他人需求 · 和谐关系 · 具体关怀 · 社群连接

**信息组织方式**：关系关怀式——以「对谁有什么影响」为线索组织信息，每个内容单元都落到具体的人和场景，温暖且具体。

---

## 核心美学

温暖的关系关怀图。像一张「这些人需要什么，我们能给什么」的社群地图——以人为中心，具体而温暖，每个角色都被看见。
比 ISFJ 更「以人为中心」（ISFJ 关注步骤，ESFJ 关注谁受影响）；比 ENFJ 更「具体落地」（ENFJ 关注成长旅程，ESFJ 关注当下需求）。

---

## 设计 Token

```
--bg:        #FEFBF8   /* 极暖白 */
--surface:   #FDF4EE   /* 浅桃，温暖中性 */
--surface-b: #F9EDE4   /* 深一档桃色 */
--text:      #2A1C14   /* 深暖棕 */
--text2:     #8A7060   /* 次要文字 */
--border:    rgba(150,80,40,0.10)
--rose:      #C4566A   /* 玫瑰红，核心「人」节点 */
--rose-bg:   #FDEEF1   /* 玫瑰浅，人物卡片背景 */
--warm:      #B87333   /* 暖铜，需求标签 */
--warm-bg:   #FEF3E6   /* 暖铜浅 */
--harmony:   #5C8A6E   /* 柔绿，和谐/满足状态 */
--harmony-bg:#EEF5F1
```

颜色原则：**玫瑰红 = 人/角色，暖铜 = 需求，柔绿 = 满足/和谐**。
颜色是温暖的，没有冷色调，没有强对比。

---

## 布局原则

**以人为中心的关系清单**：

```
[核心关怀对象]
  ├── 角色 A ── 需求：…… ── 状态：满足 ✓
  │              感受：……
  ├── 角色 B ── 需求：…… ── 状态：待关注 ○
  │              感受：……
  └── 角色 C ── 需求：…… ── 状态：满足 ✓

共同感受：……
关怀结论：……
```

- 人物卡片：玫瑰红边框，每个人一张卡，有名称/角色
- 需求标签：暖铜色小标签，具体描述需求
- 状态标注：柔绿 ✓（满足）/ 暖橙 ○（待关注）
- 共同感受区：所有角色的共同体验，放在最后
- 没有「流程」，只有「人和他们的感受」

---

## 排版

- 角色名称：玫瑰红，字重 600，像社群成员的名牌
- 需求描述：具体、口语化，不用抽象词汇
- 状态标签：小圆点或小方块，颜色即状态
- 共同感受：斜体，暖棕色，稍大字号
- 收口：第二人称，关怀感，像一封给读者的暖心提示

---

## 特有视觉元素

- **人物卡片**：玫瑰红边框，内含角色/需求/感受
- **需求标签**：暖铜色小方块，inline 嵌在卡片内
- **满足状态**：柔绿 `✓`，待关注 `○`（暖橙）
- **关系连线**：角色之间有细线标注「相互影响」
- **共同感受区**：底部区块，所有角色共享的体验

---

## 收口方式

**关怀确认**，第二人称，温暖具体：

- 「每个角色的需求都被看见——这就是和谐的起点」
- 「如果你是 [角色]，这张图也许说的就是你的感受」

ESFJ 读者需要**感受到每个人都被考虑到了**，没有人被遗漏。

---

## 不适合

- 冷色调（太冷，失去关怀感）
- 抽象框架（那是 INTJ——ESFJ 要的是具体的人，不是模型）
- 强逻辑推导（那是 INTP）
- 没有「人」的信息图（ESFJ 以人为中心，不是以概念为中心）

## 与其他风格对比

| | ESFJ | ISFJ | ENFJ |
|---|---|---|---|
| 中心 | 人 / 关系 | 步骤 / 流程 | 成长 / 旅程 |
| 结构 | 人物卡片 + 关系图 | 编号步骤 | 转化路径 |
| 核心色 | 玫瑰红（人）| 柔绿（进度）| 琥珀→绿（成长）|
| 收口 | 关怀确认 | ✅ 明确完成 | 邀请前进 |
| 视角 | 「每个人都被看见」 | 「步骤完成了」 | 「你可以到达那里」 |

### 签名视觉元素（v2）— 必须实现的布局语法

**① 人物卡片（Person Card）**
- `1px var(--rose)` 玫瑰红边框，白底，`border-radius: 6px`，`padding: 14px 16px`
- 左上角：角色名（13px bold `--rose`）+ 角色标签 pill（`--rose-bg` 背景）
- 待关注角色改用 `--warm` 暖铜色系

**② 需求标签（Need Tag）**
- `--warm-bg` 背景，`--warm` 文字，10px bold，`border-radius: 3px`
- 与需求描述横排：tag + 12px `--text2` 正文

**③ 状态徽章（Status Badge）**
- 右侧对齐，pill 形，`border-radius: 12px`
- 满足：`--harmony-bg` 绿底 + `✓ 已覆盖`；待关注：`--warm-bg` 橙底 + `○ 需关注`

**④ 共同感受区（Shared Block）**
- `--surface` 浅桃背景，顶部 `2px --rose-bg` 装饰线
- 10px uppercase 标签「共同感受」+ 13px 斜体正文
- 收口必须用关怀语气，第二人称
