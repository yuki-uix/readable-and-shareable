# ENTP Style — 发散风暴

**认知偏好**：外向直觉 · 发散联想 · 论战式思维 · 拒绝收口

---

## 核心美学

放射状联想图。像一场头脑风暴的实时记录——中心是一个命题，四周辐射出无数「但是」「如果」「还有」。
没有主轴，没有层级，只有密度和能量。读者感受到的是「这个人停不下来」。

---

## 设计 Token

```
--bg:        #FAFAF8   /* 近白 */
--surface:   #F0F0EC   /* 浅灰，次要区块 */
--text:      #1A1A18   /* 近黑 */
--text2:     #7A7A72   /* 次要文字 */
--border:    rgba(0,0,0,0.12)
--spark:     #E85D00   /* 烈橙，唯一强调色 — 核心命题节点 */
--spark-bg:  #FFF3EB   /* 橙色 tint */
```

颜色原则：**近乎无色 + 一点烈橙**。烈橙只用于：
- 中心命题节点（最大、最显眼）
- 「颠覆性」观点的边框

其余节点全部灰色系，用大小和距离表达层级，不用颜色。

---

## 布局原则

**放射状 / 蜘蛛网**，不是线性流程：

```
           [子命题 B] — [延伸 B1]
              ↗
[子命题 A] ← [核心命题] → [反例 C]
              ↘              ↓
           [子命题 D]    [但是……]
              ↓
           [更多分支]
```

- 中心节点最大（烈橙），向四周辐射
- 箭头方向多元，允许双向箭头
- 节点大小 = 距中心的层级（越远越小）
- **没有收口节点**——最外层节点全部是「开放的」
- 允许「孤儿节点」：一个想法后面跟着「?」，没有连线

---

## 排版

- 节点文字：短语为主，3-6 字，不写完整句子
- 箭头标签：可有可无，有的话用「但」「如果」「→」「≠」
- 边注：允许页面空白处有散落的短语，像灵感突然出现
- 结语：**没有结语**，或者最后一个节点是一个反问

---

## 特有视觉元素

- **中心大节点**：烈橙底色，白字，圆角，最显眼
- **双向箭头**：表示「互相矛盾」或「互为前提」
- **孤儿节点**：虚线圆形，内部 `…?`，表示「还没想完」
- **密度梯度**：中心密，边缘稀，但边缘节点更有趣
- **无收口**：故意不画一个「结论」节点

---

## 收口方式

**拒绝收口**。如果非要有一句话，是：

- 「以上都成立，但在不同前提下」
- 一个把所有分支都推翻的反问
- 空白——直接结束，没有总结

ENTP 读者不需要结论，他们需要**足够多的分支让他们自己继续想**。

---

## 不适合

- 线性流程（那是 ISFJ）
- 强结论（那是 ENTJ / INTJ）
- 单一叙事弧线（那是 INFP）
- 颜色过多（会和密度叠加，变成视觉噪音）

---

## 工具分配

**HTML + 截图**

放射状布局用 CSS `position: absolute` + 手动坐标实现，或用 SVG。
ENTP 的「密度」和「孤儿节点」用 HTML 精确控制比 AI 生成可靠。

## 与其他风格对比

| | ENTJ | ENTP | INTP |
|---|---|---|---|
| 入口 | 结论先行 | 命题爆炸 | 命题推导 |
| 结构 | 矩阵/对比 | 放射网状 | 分支推导链 |
| 收口 | 强行动指令 | 无收口 | 条件结论 |
| 强调色 | 深绿（执行）| 烈橙（能量）| 暖黄（逻辑标注）|
