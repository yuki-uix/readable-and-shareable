# ISFJ Style — 步骤手册

**认知偏好**：具体细节 · 步骤清晰 · 安全感 · 服务他人

---

## 核心美学

温暖的操作手册。像一份精心准备的 SOP——每一步都有编号，每个状态都有说明，读完知道该做什么。
没有抽象框架，没有开放问题，只有「下一步是什么」。

---

## 设计 Token

```
--bg:        #FDFCF8   /* 暖白，接近米色 */
--surface-a: #EEF3EE   /* 苔藓绿浅，完成/成功区 */
--surface-b: #F5F0E8   /* 暖米色，说明/注意区 */
--surface-c: #EEF1F8   /* 冷蓝浅，步骤序号底色 */
--text:      #242018   /* 深暖棕 */
--text2:     #7A7268   /* 次要文字 */
--border:    rgba(0,0,0,0.10)
--step:      #4A7C59   /* 苔藓绿，步骤编号 + 完成状态 */
--warn:      #B5622A   /* 暖橙，注意事项 */
```

颜色原则：**柔和、安全、可预期**。绿色 = 进度/完成，橙色 = 注意，蓝浅 = 步骤序号。
没有冷色调蓝，没有高饱和色，没有「让人紧张」的颜色。

---

## 布局原则

**垂直时间线 / 编号清单**，严格线性：

```
① 步骤一标题
   说明文字……
   ✓ 完成条件

② 步骤二标题
   说明文字……
   ⚠ 注意事项

③ 步骤三标题
   ……

✅ 完成
```

- 编号序列严格连续，绝不跳跃
- 每步结构一致（标题 + 说明 + 完成条件/注意）
- 允许「检查清单」子项：`□` 未完成，`✓` 已完成
- 「注意事项」用暖橙色方块，视觉区分于正常步骤

---

## 排版

- 标题：清晰，行动动词开头（「设置…」「确认…」「打开…」）
- 正文：完整句子，不省略主语，不用行话
- 步骤编号：绿色圆形或方形底，白色数字
- 完成条件：`✓` 开头，绿色
- 注意事项：`⚠` 开头，暖橙色底块
- 行距宽松（1.7），不压迫

---

## 特有视觉元素

- **进度指示**：顶部或侧边有步骤总数（「共 5 步」）
- **完成状态**：最后一步用绿色 `✅ 完成` 明确收口
- **注意块**：暖橙色，`⚠ 注意：` 开头，独立区块
- **检查清单**：`□` 方框，可打勾，安全感的来源
- **时间估算**：每步旁边可标注「约 X 分钟」

---

## 收口方式

**明确完成**。最后一步之后有一个绿色「✅ 完成」状态，告诉读者「到这里就结束了」。
ISFJ 读者需要知道**何时算完成**——开放结尾会让他们不安。

---

## 不适合

- 抽象框架和模型（那是 INTJ）
- 放射状发散（那是 ENTP）
- 开放收口（那是 INFP / INTP）
- 高对比强色（太有压迫感）
- 密集信息（会破坏安全感）
