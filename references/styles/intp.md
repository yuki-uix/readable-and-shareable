# INTP Style — 数学草稿

**认知偏好**：第一原理 · 精确定义 · 条件分支 · 开放收口

---

## 核心美学

手稿感系统图。像一页数学推导草稿——精确但不整齐，有划掉的线、边注、条件标签。
读者感受到的是「这个人在认真想清楚一件事」，而不是「这个人在给你一个答案」。

---

## 设计 Token

```
--bg:        #F9F9F7   /* 近白，接近草稿纸 */
--surface:   #EFEFED   /* 浅灰，次要区块 */
--text:      #1A1A18   /* 近黑，铅笔色 */
--text2:     #7A7A72   /* 次要文字，淡铅笔 */
--border:    rgba(0,0,0,0.15)
--mark:      #E8B84B   /* 暖黄，唯一强调色 — 仅用于关键定义/条件标签 */
--strike:    rgba(0,0,0,0.25)  /* 划掉的线，表示「被否定的假设」*/
```

颜色原则：**近乎无色**。暖黄 `--mark` 是唯一允许的彩色，且只用于：
- 关键定义的下划线
- 条件标签（「当 X 时」「除非 Y」）
- 被推翻的假设（配合 strikethrough）

禁止使用情绪色块。颜色 = 逻辑标注，不是情感表达。

---

## 布局原则

**推导链 + 分支**，不是单一流程：

```
命题 / 初始假设
  ↓
拆解 A            拆解 B（反向）
  ↓                   ↓
子命题 A1         反例 / 边界条件
  ↓
∴ 条件结论（在 X 假设下成立）
  ↓
? 未解问题 / 下一个假设
```

- 允许出现被划掉的节点（表示「最初以为这样，但不对」）
- 边注（margin notes）：页面右侧或左侧的细小补充，10px，灰色
- 条件标签：结论旁边用黄色小标签注明成立条件
- 箭头：直线为主，偶尔带「?」的虚线表示待验证路径

---

## 排版

- 标题：等宽字体感（Courier / 思源等宽），或普通字体不加修饰
- 正文：小号紧凑，行距适中（1.5）
- 强调：黄色下划线（定义）或 strikethrough（被推翻）
- 边注：右侧竖排或斜排，10px，text2 灰色
- 结语：条件句或问句，不是结论

---

## 特有视觉元素

- **Strikethrough 节点**：某个假设被推翻，保留但划掉，旁边注「∵ 反例是……」
- **条件标签**：黄色小方块，内容如「当 IA = landing page」「除非混合场景」
- **边注**：页面边缘的补充说明，像教材页边空白里写的笔记
- **∴ / ∵ / ≠ 符号**：逻辑符号替代箭头标签
- **问号节点**：虚线边框，表示「这条路径还没验证」

### 签名视觉元素（v2）— 必须实现的布局语法

**① 双栏布局：主内容 + 边注 gutter**
- CSS Grid `grid-template-columns: 1fr 136px`，边注 gutter 是独立一列
- 边注每条有分类 tag（`定义` / `关键` / `边界` / `待验证`），10px monospace 正文
- `待验证` 类边注用 `--mark` 黄色左边框，其他用灰色
- 边注内容与主流内容位置对应（不是统一堆在底部）

**② Strikethrough 命题（页面第一个内容块）**
- 被推翻的直觉命题，monospace，背景 `rgba(0,0,0,0.04)`，`::after` 画横线
- 紧跟 `∵ ` 开头的推翻理由（11px 灰色 monospace）

**③ 推导链 + ★ 关键变量节点**
- 每步：左侧 `①②★` 序号 + 右侧 card
- ★ 节点用 `--mark` 黄色边框，视觉突出于其他节点
- 步骤之间：`↓` + 小字标注说明跳转逻辑

**④ 条件标签行**
- `成立条件：` 前缀 + 黄色 pill（`border: 1px solid --mark`，背景 `--mark-pale`）
- 不成立条件加 strikethrough + `opacity: .6`

**⑤ 条件结论块**
- 左侧 3px `--mark` accent bar，背景 `--mark-pale`
- `∴ 条件结论` 标注 + 主文 + `当 X ∧ Y 时成立` 副文（monospace）

**⑥ 开放问题块（最后一个元素）**
- `1.5px dashed --border` 虚线边框
- `// 未解问题` 标注（monospace）
- 每条 `? ` 前缀（`--mark` 黄色加粗）

## 收口方式

**条件限定 + 开放问题**：

- 「在 [X 条件] 下，[结论] 成立」
- 「但如果 [Y 变量] 改变，结论未知」
- 最后一个节点总是问号节点

INTP 读者不需要被给出答案，他们需要看到**推导过程是严谨的**，然后自己继续想。

---

## 不适合

- 情绪色块（那是 INFP）
- 强结论收口（那是 INTJ）
- 手绘有机线条（太模糊，INTP 需要精确）
- 彩色多色系（干扰逻辑标注的语义）
- 过度留白（INTP 密度高，边注和分支会填满空间）

## 工具分配

**HTML + 截图**（同 INTJ）

精确几何 + 等宽字感 + strikethrough + 条件标签 = HTML/CSS 强项。
AI 生成无法可靠渲染 strikethrough、边注排版、条件标签位置。

## 与其他风格对比

| | INTJ | INTP | INFP |
|---|---|---|---|
| 温度 | 冷 | 冷 | 暖 |
| 线条 | 精确几何 | 精确 + 草稿感 | 手绘有机 |
| 颜色 | 冷蓝（语义） | 灰度 + 暖黄（逻辑标注） | 4色情绪色块 |
| 密度 | 高，强层级 | 高，多分支边注 | 低，大留白 |
| 收口 | 强结论 | 条件结论 + 问号 | 开放邀请 |
| 特有元素 | 蓝色框架区 | Strikethrough + 边注 + 条件标签 | 人物插图 + 波浪线 |
| 工具 | HTML | HTML | AI 生成 |
