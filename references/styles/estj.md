# ESTJ Style — 执行流程图

**认知偏好**：流程管控 · 标准执行 · 责任分配 · 结果验收

---

## 核心美学

清晰的执行流程图。像一份项目管理看板——谁负责什么、什么时候完成、完成标准是什么。
比 ENTJ 更「落地」（ENTJ 关注战略判断，ESTJ 关注具体执行）；比 ISTJ 更「外向」（ISTJ 归档规则，ESTJ 推动执行）。

---

## 设计 Token

```
--bg:        #F7F8FA   /* 冷白，略带蓝调 */
--surface:   #EEF0F4   /* 浅蓝灰 */
--surface-b: #E4E7EE   /* 深一档 */
--text:      #1A1C22   /* 近黑，略带蓝调 */
--text2:     #6A6E7A   /* 次要文字 */
--border:    rgba(0,0,40,0.12)
--navy:      #1E3A5F   /* 深海蓝，责任人标注 + 流程节点 */
--navy-bg:   #EBF0F8   /* 蓝色 tint */
--done:      #2D6A4F   /* 深绿，完成状态 */
--pending:   #B5622A   /* 暖橙，待处理 / 注意 */
```

颜色原则：**深海蓝 = 责任 / 流程主轴**，绿色 = 完成，橙色 = 待处理。
三色各司其职，不交叉。

---

## 布局原则

**横向流程 + 责任矩阵**：

```
流程主轴：
[阶段 1] → [阶段 2] → [阶段 3] → [完成]

责任矩阵：
       阶段 1    阶段 2    阶段 3
负责人   A         B         A
标准     ……        ……        ……
验收     ……        ……        ……

待处理事项：
⚠ [待处理 1]
⚠ [待处理 2]
```

- 流程主轴在顶部，横向箭头连接各阶段
- 责任矩阵列出「谁负责 / 完成标准 / 验收条件」
- 待处理事项用橙色标注
- 完成状态用绿色标注

---

## 排版

- 流程节点：深海蓝底色，白字，圆角方块
- 阶段箭头：深海蓝，细实线
- 矩阵标题行：深海蓝底，白字，小号全大写
- 待处理：`⚠` 开头，橙色
- 完成：`✓` 开头，绿色
- 结论：「按流程执行，验收标准已明确」

---

## 特有视觉元素

- **横向流程条**：顶部横跨全宽，蓝色节点 + 箭头
- **责任矩阵**：表格形式，负责人 / 标准 / 验收三行
- **状态标签**：蓝色=进行中，绿色=完成，橙色=待处理
- **验收标准行**：每个阶段明确「完成的定义是什么」
- **无模糊表达**：不出现「应该」「可能」，只有「负责人是 X」「完成标准是 Y」

---

## 收口方式

**执行确认**，责任到人：

- 「负责人已明确，按流程执行」
- 「验收标准：[具体标准]」
- 不用「建议」「考虑」——只有「执行」「验收」

ESTJ 读者不需要愿景，他们需要**清楚知道谁做什么、什么算完成**。

---

## 不适合

- 开放收口（那是 ENTP / INFP）
- 抽象框架（那是 INTJ——ESTJ 要的是具体执行，不是模型）
- 情绪色彩（那是 INFP）
- 暖色调（失去组织感）

## 与其他风格对比

| | ESTJ | ENTJ | ISTJ |
|---|---|---|---|
| 关注点 | 具体执行 / 责任分配 | 战略判断 / 结论驱动 | 规则归档 / 历史依据 |
| 结构 | 流程图 + 责任矩阵 | 结论 + 对比矩阵 | 规则表 + 来源注 |
| 核心色 | 深海蓝（组织）| 深森绿（执行）| 石板灰（规则）|
| 收口 | 责任到人 · 验收标准 | 立即行动 | 依规执行 |
