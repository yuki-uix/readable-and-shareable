# ESTP Style — 战术简报

**认知偏好**：当下行动 · 高刺激 · 实用逻辑 · 即时判断

---

## 核心美学

高对比战术简报。像一份赛前战术板——现在的局面是什么，关键变量是什么，下一步怎么打。
不是历史分析，不是长期战略，只有「现在」和「下一步」。

---

## 设计 Token

```
--bg:        #F5F5F3   /* 近白 */
--surface:   #EAEAE7   /* 浅灰 */
--text:      #1A1A18   /* 近黑 */
--text2:     #7A7A72   /* 次要 */
--border:    rgba(0,0,0,0.14)
--hot:       #D42A00   /* 高饱和红，关键变量 + 警示 */
--hot-bg:    #FFF0EE
--now:       #1A1A18   /* 近黑大字，当前局面 */
```

颜色原则：**高对比黑白 + 高饱和红**。红色标注关键变量和即时风险，不用于装饰。

---

## 布局原则

**局面 → 关键变量 → 下一步行动**，时序紧凑：

```
[当前局面 — 大字，最显眼]

关键变量：
  ⚡ [变量 A] — 影响：……
  ⚡ [变量 B] — 影响：……

现在：[当前状态]
下一步：[立即可执行的动作]
```

- 当前局面：最大字号，占页面顶部
- 关键变量：`⚡` 标注，红色高亮
- 「下一步」：粗黑字，动词开头，立即可执行
- 没有背景故事，没有「原因分析」——只有现状和行动

---

## 特有视觉元素

- **大字局面描述**：顶部最显眼，字号最大
- **⚡ 关键变量**：红色标注，每个变量后面是「影响」
- **时间感**：「现在」「下一步」「立即」等词强调即时性
- **无历史回顾**：不写「之前是……」，只写「现在是……」

### 签名视觉元素（v2）— 必须实现的布局语法

**全局：深色主题**
```
--bg:      #111110   /* 近黑 */
--surface: #222220   /* 卡片背景（比 bg 浅，保证层次感）*/
--text:    #F2F2EF   /* 主文字，高对比 */
--text2:   #A8A8A2   /* 次要文字（≥ 6.5:1 对比度，不得低于此值）*/
--text3:   #6E6E68   /* 纯装饰 label，全大写小字 */
--red:     #E03A2F
```
**注意**：深色主题下 `--text2` 的最低对比度要求是 6.5:1（对应 `--surface`）。不得使用 #888882 或更暗的灰色作为内容文字。

**① 红色 Header Strip**
- 全宽红色背景（`--red`），左侧「风格标签 · 全大写」，右侧日期/文件类型
- 文字用 `rgba(white, 0.8)` 保证可读性

**② 局面块（Situation）**
- `局面` 小标注（10px，`--text3`）
- 大字标题：26px，font-weight 800，letter-spacing -.02em
- 关键词用 `<em>` + `--red` 色突出（`font-style: normal`）

**③ 关键变量块（KeyVar）**
- 左侧 4px `--red` accent bar，背景 `--red-pale`（`#311210`）
- `关键变量` 标注（10px，`--red`）+ 14px bold 主文 + 11px 灰色副文

**④ NOW / NEXT 二列切割**
- CSS Grid `grid-template-columns: 1fr 1px 1fr`，中间 1px 分割线
- NOW 列：标注红色点 + `NOW`（`--red`）；NEXT 列：灰色点 + `NEXT`（`--text3`）
- 每列内：编号方块（`NOW` 用红底白字，`NEXT` 用 `--surface2` 底）+ 行动文字

**⑤ 摩擦场景小卡网格**
- 2×2 网格，每卡：场景名称（11px bold）+ `× [错误做法]`（红色 `×` 前缀）+ `→ [正确做法]`（灰色 `→` 前缀）
- 卡片背景 `--surface`，圆角 4px，无边框

**参考实现**：`docs/preview-estp-v2.html`（2026.06）

## 收口方式

**立即行动**，强时间感：

- 「下一步：[具体动词短语]」
- 不解释，不回顾，直接给出下一个动作

---

## 不适合

- 长期战略分析（那是 ENTJ/INTJ）
- 情感叙事（那是 INFP）
- 历史依据（那是 ISTJ）
- 大量文字说明
