# INFJ Style — 意义地图

**认知偏好**：整体观 · 意义连接 · 深度优先 · 隐喻式表达

**信息组织方式**：意义连接式——展示概念之间如何相互召唤，没有步骤和对比，只有「A 指向 B，B 转化为 C」的深层关系网。

---

## 核心美学

概念关系网。像一张手绘的「意义地图」——不展示步骤，不展示对比，而是展示**概念之间如何相互召唤**。
读完不是「我知道怎么做了」，而是「我看见了一个更大的图景」。

---

## 设计 Token

```
--bg:        #F7F5FB   /* 极浅紫白，冷暖之间 */
--surface:   #EEEAF5   /* 浅紫，次要区块 */
--text:      #1C1828   /* 深紫黑 */
--text2:     #7A6E8A   /* 紫灰，次要文字 */
--border:    rgba(80,60,120,0.12)
--core:      #5B3FA6   /* 深紫，核心概念节点 */
--core-bg:   #EDE8F8   /* 紫色 tint */
--link:      #8B6CC8   /* 中紫，关系连线 + 次要节点 */
--gradient-start: #5B3FA6  /* 渐变起点，深紫 */
--gradient-end:   #2A7A8A  /* 渐变终点，深青 */
```

颜色原则：**深紫 → 深青渐变**，表达「深度」和「转化」。
- 深紫 = 核心概念（最重要的节点）
- 中紫 = 次要概念节点
- 渐变连线 = 概念之间的「意义流动」
- 灰紫底 = 补充说明区

---

## 布局原则

**概念网络 / 关系图**，不是树状或线性：

```
  [概念 A] ──意义流──→ [核心概念]
       ↑                    ↓
  [概念 C] ←──隐喻──── [概念 B]
       |
  [更深的层]
```

- 核心节点居中，其他节点围绕（不一定对称）
- 连线有标签：「召唤」「隐含」「转化」「矛盾中共存」
- 允许概念节点有「副标题」解释它的深层含义
- 渐变色连线：从深紫流向深青，表示意义的流动方向
- 留白适中——不是 INFP 那种大留白，但也不是 ENTJ 的高密度

---

## 排版

- 核心节点：深紫底色，白字，圆角，略大
- 次要节点：浅紫底色，深紫字
- 连线标签：10px 斜体，紫灰色
- 补充说明：浅紫灰底块，11px，行距 1.8
- 标题：深紫黑，不全大写，有诗意感
- 结语：一句话，隐喻式，指向整体而非细节

---

## 特有视觉元素

- **渐变连线**：从深紫渐变到深青，用 SVG 或 CSS gradient border
- **副标题节点**：主标签下有一行小字解释「这个概念真正说的是……」
- **整体标注**：图的角落有一句话描述「整张图在说什么」
- **矛盾共存**：允许两个节点之间有「←→ 矛盾中共存」的双向标注
- **深度层**：某些节点下面有「↓ 更深：……」的延伸线，表示还有更多

---

## 收口方式

**整体感悟**，不是行动指令，不是结论句。收口是：

- 一句让人停下来想的话
- 一个指向「更大图景」的句子
- 「这张图本身就是答案」的沉默

INFJ 读者不需要被告知「结论是 X」，他们需要**感受到整个系统在说同一件事**。

---

## 不适合

- 强行动指令（那是 ENTJ）
- 步骤化流程（那是 ISFJ）
- 纯逻辑推导（那是 INTP）
- 低密度大留白（INFJ 需要足够的概念节点来构建「图景感」）
- 高饱和色（破坏深度感）

### 签名视觉元素（v2）— 必须实现的布局语法

**① 核心概念圆（Core Circle）**
- 圆形（64×64px），深紫背景（`--core: #5B3FA6`），白字核心词
- 位于卡片上方视觉中心，通过 flex 居中
- 下方浮动小标注「核心概念」（9px，`rgba(white,0.6)`，紧贴圆底）

**② 意义连线（Meaning Link）**
- 用 CSS `border-left + border-top + transform: rotate` 实现 L 形连线，或直接用 flex 列间加连接线
- 每条连线旁附关系词 pill：白色背景，`--border` 边框，8px 圆角，10px 字号，如「召唤」「转化」「指向」
- 线条颜色：从 `--core` 渐变到 `--link`

**③ 卫星节点（Satellite Node）**
- 圆形（44×44px），`--core-bg` 背景，`--core` 文字和 1px 边框
- 2-4 个，围绕核心概念排列（可用 flex row 模拟）
- 节点下方一行说明（11px，`--text2`，最多 2 行）

**④ 整体洞见收口（Insight Footer）**
- 全宽渐变背景（`--gradient-start` → `--gradient-end`），白字
- 顶部小标注「整体洞见」（9px，`rgba(white,0.6)`），下方感悟句（13px bold white）
- 无边框，直接接地，视觉上「封印」整张卡片
