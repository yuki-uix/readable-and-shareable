# ISTJ Style — 规则档案

**认知偏好**：历史依据 · 规则归档 · 精确执行 · 先例优先

**信息组织方式**：档案归纳式——每条信息都有编号、来源和标准，像一份权威的参考手册，先例优先，以规则和历史依据建立信任。

---

## 核心美学

冷静的参考手册。像一份精心维护的内部规范文档——每条规则都有来源，每个判断都有依据，读完知道「这件事的标准是什么」。
不像 ISFJ 的温暖步骤，ISTJ 更冷、更「权威」，像一份官方文档而不是操作指南。

---

## 设计 Token

```
--bg:        #F8F8F7   /* 冷灰白 */
--surface:   #EEEEED   /* 浅石板灰 */
--surface-b: #E4E4E2   /* 深一档的灰，规则区 */
--text:      #1C1C1A   /* 近黑 */
--text2:     #72726A   /* 次要文字 */
--border:    rgba(0,0,0,0.14)
--rule:      #3D3D3A   /* 深灰，规则编号 + 权威标注 */
--source:    #6B6B62   /* 来源标注，比正文浅 */
```

颜色原则：**近乎无彩色**。不用强调色，用字重和层级建立权威感。
- 深灰 = 规则编号、标题、权威来源
- 中灰 = 正文、说明
- 浅灰 = 底色区块

---

## 布局原则

**规则表 + 来源注释**，参考文档结构：

```
[文档标题]
[适用范围 / 前提条件]

规则 1.1  [规则内容]
          来源：[先例/依据]

规则 1.2  [规则内容]
          例外：[例外情况]

规则 2.0  [更高层规则]
  规则 2.1  子规则
  规则 2.2  子规则

[适用结论：依规执行]
```

- 编号系统：1.0 / 1.1 / 1.2 / 2.0 等层级编号
- 每条规则后有「来源」或「依据」的小字注释
- 例外情况单独标注，不省略
- 没有「推荐」「建议」——只有「规定」「依据」

---

## 排版

- 标题：等宽字体感，全大写小字标签
- 规则编号：深灰，字重 700，等宽
- 规则内容：字重 500，正文色
- 来源注释：小号（10px），source 色，缩进
- 结论：「依规执行」，不是「我建议」

---

## 特有视觉元素

- **层级编号系统**：1.0 / 1.1 风格，像法律文本或技术规范
- **来源注释**：每条规则下方的灰色小字，标注「先例：…」「依据：…」
- **例外块**：缩进的「例外：」段落，单独处理
- **适用范围框**：顶部区块，说明「本规则适用于…」
- **无颜色层级**：完全靠字重、字号、缩进建立层级

---

## 收口方式

**规则确认**，无情感色彩：

- 「依照以上规则执行」
- 「如有例外，参见 [来源]」
- 不用「建议」「推荐」「也许」——只有规定

ISTJ 读者不需要被说服，他们需要**看到规则有据可查、来源可信**，然后执行。

---

## 不适合

- 暖色调（失去冷静权威感）
- 开放收口（ISTJ 需要明确的规定，不是邀请）
- 手绘感（不够严谨）
- 情绪化表达（不在档案文档里）
- 过多留白（浪费空间，ISTJ 偏好信息密度）

### 签名视觉元素（v2）— 必须实现的布局语法

**① 规则编号卡（Rule Card）**
- 左上角规则编号（`R-001` 格式，12px bold `--rule`，`font-family: mono`）
- 正文规则（13px，`--text`，行距 1.6）
- 右下角来源标注（10px `--source`，如「来源：官方文档」）
- 卡片背景 `--surface`，左侧 3px `--rule` accent bar，`border-radius: 3px`

**② 权威区块（Authority Block）**
- 背景 `--surface-b`（深石板灰），白色/浅灰文字
- 浮动标签「规范」（9px 全大写，mono，`rgba(white,0.5)`）
- 主文 14px bold，副文 11px `rgba(white,0.7)`
- 用于「这是硬性规定」的条目，视觉上与普通规则明确区分

**③ 先例引用行（Precedent Row）**
- 左侧「先例：」（10px mono `--source`），右侧「标准：」（10px mono `--text2`）
- 中间 1px 分隔线
- 用于「从历史依据推出当前标准」的逻辑

**④ 归档状态标注（Archive Tag）**
- 右上角小 pill：「已归档」/ 「现行」/ 「待审核」（9px，不同背景色）
- 「已归档」= `--surface-b` 灰底；「现行」= 浅绿底；「待审核」= 浅橙底

## 与其他风格对比

| | ISTJ | ISFJ | ENTJ |
|---|---|---|---|
| 权威来源 | 历史先例 / 规则 | 步骤经验 | 战略判断 |
| 结构 | 规则表 + 来源注 | 编号步骤 | 结论 + 矩阵 |
| 色调 | 冷石板灰（无彩）| 柔暖绿（情感）| 深森绿（执行）|
| 收口 | 依规执行 | ✅ 明确完成 | 立即行动 |
| 温度感 | 冷 | 暖 | 中性偏强硬 |
