# Digest Framework — /digest Skill

深度解读产物 `digest.md` 的 schema 定义与写作标准。

## 输出路径

```
examples/{slug}/digest.md
```

**Slug 规则**：2-4 词，kebab-case，取自文章标题或 URL。
**覆盖策略**：每次执行直接覆盖，不备份。

---

## Schema

```markdown
# Digest — {文章标题}

**来源**：{URL 或文件路径}
**日期**：{YYYY-MM-DD}

---

## 核心主张

作者真正在说什么？一句话，必须是论点而不是话题。
> 格式示例："作者认为 X，因为 Y，因此 Z。"

---

## 隐藏假设

这个论点成立，需要读者接受哪些前提？（2-3 条）

- {假设 1}
- {假设 2}
- {假设 3，如有}

---

## So What

读完这篇文章，什么认知需要更新？
不是「学到了什么」，而是「之前的什么判断是错的或不完整的」。

---

## 盲点 / 反例

这个结论在什么条件下不成立？
作者有意或无意回避了什么问题？

---

## 更大的连接

这篇文章指向的更大趋势或问题是什么？
它和哪些已有框架 / 概念形成对话或张力？
```

---

## 写作标准

| 标准 | 要求 |
|---|---|
| **不是摘要** | 不复述原文说了什么，只说它意味着什么 |
| **有观点** | 每个部分都有可被质疑的判断，不是中性描述 |
| **够具体** | 隐藏假设和盲点必须针对这篇文章，不套通用说法 |
| **篇幅** | 300-500 字，读来像一篇有密度的读书笔记 |
| **语言** | 中文；英文文章的解读也用中文写 |

---

## 与旧 analysis.md 的区别

| | `analysis.md`（旧）| `digest.md`（新）|
|---|---|---|
| 问题 | 文章的结构是什么 | 文章的意义是什么 |
| 输出 | 结构类型 + 核心论点列表 | 可被质疑的解读判断 |
| 用途 | 推荐输出类型 | 作为卡片内容的来源 |
| 价值 | 任何 AI 都能做 | 需要真正读懂 + 有观点 |
